# Private Registries

Some projects require access to private package registries like GitHub Packages. Check out the [official documentation for how to authenticate](https://docs.github.com/en/packages/guides/package-client-guides-for-github-packages) against these registries.

## Authenticating

We configure access to package registries on a global level and not within the repository

- Check out how to use a [global NuGet config](https://docs.microsoft.com/en-us/nuget/consume-packages/configuring-nuget-behavior)
- Check out how to use a [per-user NPM config](https://docs.npmjs.com/cli/v7/configuring-npm/npmrc#per-user-config-file)

### Generate a personal access token

You need to generate a PAT to read the npm packages. In order to do that, follow this steps:

1. Go to <https://github.com> and click your profile on the top right and select _Settings_

   ![User settings](/img/docs-internal/conventions/githubUserSettings.png)

2. Select _Developer settings_ in the menu on the left side

   ![Developer settings](/img/docs-internal/conventions/githubDeveloperSettings.png)

3. Select _Personal access token_ in the menu on the left and click the _Generate new token_ button on the top right

   ![Personal access token](/img/docs-internal/conventions/pat.png)

4. Type **NPM Token** in the _Note_ input and check the _read:packages_ scope and finally click the _Generate token_ button

   ![npm token](/img/docs-internal/conventions/npmTokenGithub.png)

5. Copy the generated PAT to the clipboard

   ![generated PAT](/img/docs-internal/conventions/generatedPat.png)
